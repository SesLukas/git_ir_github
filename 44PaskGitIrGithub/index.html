Git'as kodo versijavimo programa Github'as versijuoto kodo talpykla internete

Git komandos

  • git init sukurti git repozitoriją dabartiniame aplankale
  • git status pateikia git repozitorijos dabartinę info
  • git add prideda nurodytus failus prie commitinimui paruoštų failų
  • git rm --cached || restored --staged išima nurodytus failus iš stafe'inimo zonos(unstage'ina)
  • git commit -m "žinutė apie commitą" sukuria kodo versiją iš stage'intų failų ir prideda žinutę apie pakitimus(trumpa ir aiški,ne daugiau 50 simbolių)
  • git restore sugrąžina nurodytą/us failą/us į pradinęs būseną, jeigu jie nebuvo add'int į stage'ingo zoną
  • git push nusiunčia lokalios repozitorijos commit'us į išorinę repozitoriją
  • git pull atsiunčia išorinės repozitorijos pakitimus į lokalią repozitoriją
  • git log išveda commit'ų sąrašą
  • git clone nukonuoja nurodytą projektą į dabartinį aplankalą
  • git checkout commit|branchName pereinama į nurodytą kodo versiją arba kitą šaką
  • git branch NaujosSakosPavadinimas sukuriama nauja šaka
  • ...

Branch protection rules

Vienas svarbiausių šakų apsaugos taisyklių yra "Pull Request"(bandymas vienos šakos commit'us sukelti į kitą šaką(merge))
Pull Request taisyklės viduje dar reiktų suteikti "Require approvals" nustatymą, kuris reikalauja, kad nurodytas kiekis kitų projekto daluyvių patvirtintų tą Pull Request'ą, norint, kad šakos būtų apjungtos
